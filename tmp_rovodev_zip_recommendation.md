# ZIP 배포 방식 - 최종 답변

## 질문에 대한 답변

**"ZIP으로 배포하면 '확인되지 않은 개발자' 메시지가 뜨고, 우클릭 → 열기로 실행 가능한가?"**

### ✅ 답변: 네, 가능합니다!

## 실제 동작 방식

### ZIP 배포 시나리오

```
1. 사용자가 ZIP 다운로드
   ↓
2. ZIP 압축 해제 (더블클릭)
   ↓
3. Octopus.app을 Applications로 이동
   ↓
4. 앱 실행 시도 (더블클릭)
   ↓
5. Gatekeeper 검증
   ↓
6. 메시지 표시:
   - "확인되지 않은 개발자" 또는
   - "손상되었습니다"
   ↓
7. 우클릭 → "열기" 선택
   ↓
8. ✅ 실행 성공!
```

## DMG vs ZIP - 실제 차이

### 사용자 경험 측면

| 측면 | DMG | ZIP |
|------|-----|-----|
| **설치 단계** | 마운트 → 드래그 → 언마운트 | 압축 해제 → 이동 |
| **UI** | 드래그 앤 드롭 화면 | 없음 |
| **첫 실행** | 우클릭 → 열기 | 우클릭 → 열기 |
| **프로페셔널** | ✅ 높음 | ⚠️ 보통 |

### Gatekeeper 검증

**중요**: 이론적으로는 동일하지만, 실무 경험상:

- **DMG**: macOS가 더 엄격하게 검사 (표준 포맷이므로)
  - "손상됨" 메시지 나올 확률 높음
  
- **ZIP**: 범용 압축 포맷
  - "확인되지 않은 개발자" 메시지 나올 확률 높음
  - 우클릭 → 열기가 더 잘 작동하는 경향

## 권장사항

### 상황 1: 사용자 편의 최우선

**선택**: DMG
- 프로페셔널한 설치 경험
- 드래그 앤 드롭 UI
- 단, 일부 사용자는 "손상됨" 메시지 볼 수 있음

### 상황 2: 실행 성공률 최우선

**선택**: ZIP ⭐ (추천!)
- "확인되지 않은 개발자" 메시지 (더 친숙함)
- 우클릭 → 열기가 더 잘 작동
- 단, 수동 설치 필요

### 상황 3: 양쪽 모두 제공

**선택**: DMG + ZIP 둘 다 배포
```json
"target": ["dmg", "zip"]
```
- 사용자가 선택 가능
- 권장!

## 우리의 결정

### 추천: ZIP 배포 (또는 DMG + ZIP)

**이유**:
1. ✅ "우클릭 → 열기" 성공률이 더 높음
2. ✅ "확인되지 않은 개발자" 메시지가 덜 무서움
3. ✅ 간단한 설치 과정
4. ✅ 파일 크기 약간 작음

**단점**:
- ⚠️ 수동으로 Applications 폴더로 이동 필요
- ⚠️ 드래그 앤 드롭 UI 없음

## 설정 변경

### package.json - ZIP만

```json
{
  "mac": {
    "target": ["zip"],           // ZIP만
    "identity": "-",
    "hardenedRuntime": true,
    "entitlements": "build/entitlements.mac.plist"
  }
}
```

### package.json - 둘 다 (권장!)

```json
{
  "mac": {
    "target": ["dmg", "zip"],    // 둘 다 제공
    "identity": "-",
    "hardenedRuntime": true,
    "entitlements": "build/entitlements.mac.plist"
  }
}
```
